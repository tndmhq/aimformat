# Plan: PR #17 Codex P2 parity fixes (TS reader vs Python reader)

Four Codex P2-minor findings on PR #17, all TS-reader-vs-Python-reader
divergences on **non-canonical / hand-edited** input. Ruling principle: the
parity harness pins the TS projection to the Python reader, so the TS side
moves. Python behavior was verified empirically on each exact input first.

1. **First duplicate attribute wins** (`ts/src/canonical.ts`). TS canonical
   serialization let later duplicates win; Python `canonical_attrs` uses
   `setdefault` (first wins), matching `Element.get`. Fix: first-wins in
   `canonicalAttrs`. Verified: Python serializes
   `<p data-aim="x" data-aim="y">` as `<p data-aim="x">`.
2. **Self-closing non-void tags normalize to open+close**
   (`ts/src/canonical.ts`). TS echoed `<p data-aim="x"/>` back self-closing;
   Python `serialize` has no self-closing branch outside SVG, so it emits
   `<p data-aim="x"></p>`. Fix: drop the `node.selfClosing` echo branch.
3. **Semicolonless character references decode like `html.unescape`**
   (`ts/src/parser.ts`). Python's `html.parser` (via `html.unescape`) decodes
   `&#65b` → `Ab`, `&amp b` → `& b`, `&copy 1` → `© 1`, and longest-prefix
   forms like `&ampx` → `&x`; TS kept them literal. Fix: port the
   `html._charref` regex + `_replace_charref` algorithm with the legacy
   (no-semicolon) named-entity table. Unknown *name-shaped* `&name;` refs
   keep the existing strict parse error (canonical .aim writes raw UTF-8);
   that reject-instead-of-guess divergence is deliberate and documented.
4. **Container members use the local duplicate-id chunk**
   (`ts/src/document.ts` + `scripts/dump_projection.py`). When a chunk id
   repeats under multiple containers, TS `getAll` already records per-parent
   groups but container member construction reused the global first chunk
   view, so the second container showed the first member's html/text. The
   dump script mirrored the same bug (`chunk_by_id`), while Python's actual
   per-container primitives (`container.elements()`, `_direct_members`) are
   local. Fix both sides to build member views from the local sibling group;
   existing goldens (unique ids) are byte-identical.

Each finding gets a non-canonical parity fixture (SDK-built, then
deterministic string edits) generated by `scripts/gen_parity_fixtures.py`,
its golden from `scripts/dump_projection.py`, and TS unit tests. One commit
per finding; full Python + TS gates green.
