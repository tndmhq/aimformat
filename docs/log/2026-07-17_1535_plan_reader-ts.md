# Plan: TypeScript read library + cross-implementation parity harness

Maintainer-approved scope (2026-07-17): an official TypeScript **read**
library for `.aim` inside this repo, plus a parity harness that pins the
Python SDK and the TS reader to the same projection of the same fixtures.
Writes stay Python-only.

## Deliverables

1. **`ts/` — `@aimformat/reader`** (version `0.0.0`, not published):
   dependency-free TypeScript source consumed directly (no build step;
   Vite/vitest consumers transpile). Read-only projection mirroring the
   `AimDocument` read surface: ordered top-level nodes (chunks and
   containers, recursive members), `Chunk` (id, container, member tags in
   order — runs first-class, canonical html, text), proposals, theme /
   `aim:doc` settings / resolved page setup, embedded stylesheet
   extraction, agent note, asset-registry ids, history + embeddings as
   opaque JSONL blocks, `docHash`, and by-id lookup as a multimap (repeated
   ids never overwrite).
2. **Parser**: a strict parser for the canonical HTML subset (spec §11) —
   one code path in Node and browsers, no DOMParser split; mirrors the
   Python `dom.py` transparent reader; rejects non-canonical ambiguity with
   clear errors. Trade-offs documented in `ts/README.md`.
3. **Parity harness — the spec is ground truth**:
   `scripts/gen_parity_fixtures.py` (SDK-generated edge fixtures under
   `tests/parity/fixtures/`: runs, nested containers, every proposal
   action, theme/doc blocks, packed assets, empty registry, unicode /
   whitespace edges), `scripts/dump_projection.py` (Python SDK projection →
   committed golden JSON, covering `examples/*.aim` too), a pytest that
   regeneration is byte-stable, and TS tests asserting field-for-field
   equality against the goldens. Divergences are adjudicated against
   `spec.md`; Python-side bugs get fixed in Python with regression tests.
4. **CI**: a `ts` job in `ci.yml` (npm ci, prettier check, `tsc --noEmit`,
   vitest), mirroring the existing pinned-action style.

## Gates

Python: `ruff check`, `ruff format --check`, `mypy`, full `pytest`,
`aim lint examples/*.aim`. TS: `tsc --noEmit`, vitest, `prettier --check`.

**Outcome (same day):** shipped as planned — strict canonical-subset
scanner, full read projection, 11 golden-pinned sources plus `docHash`
parity over the ok\_\* conformance kit, node CI job. All gates green.
Zero cross-implementation divergences surfaced, so no Python-side fixes
were needed; the two implementations agree byte-for-byte on canonical
html and `docHash` for every source. Durable notes promoted to
`docs/knowledge/architecture.md` (§The TypeScript reader).
