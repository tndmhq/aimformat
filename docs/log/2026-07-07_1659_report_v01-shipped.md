# Report: spec v0.1 + reference toolkit shipped

Executed the plan (including the same-day ingest/export addendum). The
repo now carries everything a developer needs to build on `.aim`.

## What shipped

- **`spec.md` v0.1** — single normative file: anatomy, substrate, chunks,
  pending lane, history, caches, assets, canonical form + `doc_hash`,
  security, conformance. Appendix A generated from the registry; every
  ` ```aim ` snippet linted in CI (one carries a real verifying checkpoint
  hash); Appendices B/C separate informative recommendations and future
  work.
- **`aimformat` 0.1.0** — stdlib-only SDK (`AimDocument`: ops, batches,
  proposals with supersede/chains/accept-with-tweaks, undo/redo,
  checkpoints, `verify()`, `state_at()`, flatten/prune, asset pack/gc,
  summary/TOC/embeddings), registry-driven verifier with stable rule
  codes, deterministic `aim.css` generator (11.2 KB raw — inside the
  spec's budget), `aim` CLI, docling ingestor, DOCX exporter with real
  tracked changes.
- **Tests: 267** across canonical form, DOM, ids, ops, proposals, history
  (invertibility, tamper detection, time travel), every lint-rule family,
  css, assets, caches, CLI, ingest/export round-trips, executable spec
  snippets, generated examples, and the committed `ok_*`/`nok_*`
  conformance suite (24 files, one rule each).
- **Docs**: README (quickstart, API table, interop), getting-started
  guide, CONTRIBUTING (dev setup + registry workflow), CHANGELOG, CI
  workflow (3.10–3.13), `.gitattributes`, `docs/knowledge/architecture.md`.

## Notable engineering facts

- The verify primitive is payload byte-equality over inverse replay on a
  deep copy — the same walk powers `state_at`. It caught every authoring
  slip during fixture writing; treat C001/H006 findings as truth.
- Ingestion lands through the public ops in one batch, so imported
  documents carry their provenance as ordinary history and verify like
  anything else.
- docling-core's exported dict labels ordered lists as plain `list`;
  orderedness is read from the items' `enumerated` flags (test-pinned).
- python-docx's `Paragraph.text` does not surface runs inside `w:ins` —
  the export tests assert revision markup at the XML level.
